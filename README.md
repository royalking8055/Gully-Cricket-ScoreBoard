This project presents two cricket dashboards designed to demonstrate both professional match visualization and real-time scoring for casual games.

The first dashboard is a Match Scorecard Dashboard that displays a complete cricket match summary in a professional editorial-style layout similar to modern sports analytics platforms. It features a dark-themed match summary hero section showing the overall score, along with a detailed batting scorecard including runs, balls faced, strike rate, boundaries, and dismissal information. A bowling section provides statistics such as overs bowled, wickets taken, runs conceded, and economy rate. The dashboard also includes run rate tracking, over information, and dot ball visualization for better match analysis. Clean typography using Bebas Neue and DM Mono fonts gives the interface a modern sports-style look. This dashboard is read-only and designed mainly for viewing a completed match with clear and structured data presentation.

The second dashboard is a Gully Cricket Scorecard, a fully interactive real-time scoring application designed for friends and classmates playing gully or backyard cricket. I created this dashboard for my friends because when we play cricket at college with a minimum number of players, we often struggle to identify or calculate the current score, wickets, and overs during the game. This tool helps track the match automatically and keeps the score accurate without manual calculations.

The application begins with a match setup screen where users can enter team names, add player names, and configure match settings such as overs and wicket limits. During the game, a live scoring screen allows users to record each ball using scoring buttons for runs (0–7), wides, no balls, byes, and wickets. The interface includes a large live score display, batter cards showing runs and strike rate, a bowler tracker, and a ball-by-ball “this over” visualizer. Additional features such as an undo button, automatic new batsman selection after a wicket, and bowler change prompts after each over help maintain accurate scoring.

Once the match ends, the result screen displays the final scorecard along with batting and bowling statistics and highlights the star performers of the match. By default, the app uses 2 overs and 1 wicket, making it ideal for quick and simple gully cricket matches.

Overall, this project demonstrates how dashboard design and simple data tracking tools can make cricket scoring more organized, accurate, and visually clear, whether for analyzing a completed match or tracking a quick game among friends.
